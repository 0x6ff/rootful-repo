Recovery instructions for ETA Checker Revamped

NOTE: These instructions will NOT work for ETA Checker Legacy as that tweak uses a different way to harm your device!

NOTE: Please make sure you have a macOS (10.15+) or Linux machine (Debian based preferably)

NOTE: These instructions will also NOT work on non-checkm8 devices (A12+). If your device is above this, restore it.

 

 

Step 1: Clone Nathan's SSHRD Script with git clone https://github.com/verygenericname/SSHRD_Script.git --recursive. Add sudo to the start of the command if using Linux.

 

Step 2: Once Git has finished cloning the repository, cd into it with cd SSHRD_Script.

 

Step 3: Run ./sshrd.sh "ios version" without the quotes and ios version being your iOS version. Add sudo to the start of the command if using Linux.

NOTE: This may take a while as it is generating a ramdisk for your device. Be patient.

 

Step 4. Run ./sshrd.sh boot to boot the SSH ramdisk. As with the previous steps, add sudo to the start of the command if using Linux.

NOTE: It is normal that you're seeing text on the screen. Ignore it.

 

Step 5. Run ./sshrd.sh ssh to gain SSH access to your device. Add sudo to the start of the command if using Linux.

 

Step 6. Run mount_filesystems /mnt1. sudo will no longer be needed after the last step.

 

Step 7. Run mv /mnt1/Trolled.plist Info.plist

 

 

Your device should now be fixed! Press CTRL+C to exit the SSH connection and then run ./sshrd.sh reboot.

If you still experience a bootloop, please ask for support in the Hephaestus/Ironside Discord server found on the homepage.
